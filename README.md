# breedy GitHub Pages 배포본

이 폴더는 별도의 빌드 과정 없이 GitHub Pages에 바로 배포할 수 있는 정적 UI 데모입니다.

## 업로드 방법

1. 이 ZIP 파일의 압축을 풉니다.
2. 압축을 푼 폴더 안의 `index.html`, `.nojekyll`, `README.md`를 GitHub 저장소 최상위 경로에 업로드합니다.
3. GitHub 저장소에서 **Settings → Pages**로 이동합니다.
4. **Build and deployment**를 `Deploy from a branch`로 설정합니다.
5. 배포 브랜치를 `main`, 폴더를 `/(root)`로 지정하고 저장합니다.

ZIP 파일 자체를 저장소에 올리는 방식이 아니라, 압축 안의 파일을 저장소 루트에 올려야 합니다.

## 접속 경로

- 사용자 앱: `https://<GitHub 사용자명>.github.io/<저장소명>/`
- 관리자 화면: `https://<GitHub 사용자명>.github.io/<저장소명>/#admin`

화면 우측 하단의 플로팅 버튼으로 사용자 앱과 관리자 화면을 전환할 수 있습니다.

## 현재 구현 범위

- 사용자 앱과 경량 관리자 백오피스 UI
- 홈, 개체, 커뮤니티, 마켓, 근처 탭
- 개체 상세, 기록 입력, 게시글 및 리스팅 작성 화면
- 신고, 검토 큐, 사용자, 리스팅, 게시글, 장소, 안전 규칙 관리 화면
- `#admin` 해시를 이용한 관리자 화면 직접 접근
- 관리자 화면의 상태 변경 및 토스트 메시지 데모

## 현재 제한사항

- 정적 프런트엔드 UI 데모이며 실제 서버와 연결되지 않습니다.
- 로그인, 관리자 권한, 데이터베이스, 파일 저장 기능이 없습니다.
- 입력하거나 변경한 데이터는 새로고침하면 초기화됩니다.
- 이미지와 웹폰트는 외부 URL을 사용하므로 인터넷 연결이 필요합니다.

실서비스에서는 인증, 데이터베이스, 이미지 스토리지를 별도로 연결해야 합니다.
